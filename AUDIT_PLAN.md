# خطة التدقيق والتنفيذ — LENA Dress Roomshow

## الهدف
تنفيذ تدقيق عملي للمستودع ثم إصلاح المشاكل الحرجة والسريعة أولاً، مع توثيق ما تم اكتشافه وما يحتاج مرحلة ثانية.

## ما تم التحقق منه مبدئياً
- تم عمل clone للمستودع بنجاح.
- تم تثبيت الاعتمادات بنجاح عبر `npm install`.
- `npm run typecheck` ✅ ناجح.
- `npm run build` ✅ ناجح.
- `npm test` ✅ جميع الاختبارات الحالية ناجحة.
- `npm run lint` ❌ كان يفشل بسبب `no-explicit-any` في `AddAppointmentModal.tsx`.

## ملاحظة مهمة
المستودع المحلي بعد الـ clone **سليم بنيوياً** ويختلف عن الانطباع الأول المأخوذ من قراءة raw files عبر الويب. لذلك سيتم الاعتماد على النسخة المحلية المستنسخة كمرجع التنفيذ.

## أولويات التدقيق

### المرحلة 1 — Critical / Release blockers
1. إصلاح أي أخطاء lint/type/build تمنع جودة التسليم.
2. فحص المسارات الجديدة والتأكد من أنها مربوطة داخل التطبيق فعلياً.
3. التحقق من أن الميزات المعلنة ليست مجرد ملفات غير مستخدمة.
4. التحقق من الثغرات/المخاطر العالية في الاعتمادات.

### المرحلة 2 — Functional bugs
1. مراجعة رفع الصور وتحديث الحالة أثناء رفع عدة صور.
2. مراجعة المواعيد وإزالة `window.location.reload()`.
3. مراجعة الباركود: هل يوجد scan حقيقي أم مجرد فتح كاميرا؟
4. مراجعة Landing Page: هل هي مرتبطة بالـ routes الرئيسية وهل هي مكتملة وظيفياً؟

### المرحلة 3 — Architecture / UX / Performance
1. تقييم localStorage مع صور Base64.
2. تقييم فصل الخدمات والأنواع وربطها مع الواجهات.
3. مراجعة قابلية التوسع والاختبار.
4. تحديد quick wins للبيع والتسليم.

## الاكتشافات الحالية

### 1) حالة الجودة الحالية
- البناء والاختبارات ناجحة.
- يوجد خلل lint واحد وتم إصلاحه.

### 2) مشاكل وظيفية ظاهرة
- `BarcodeScanner.tsx` يفتح الكاميرا فقط، ولا يوجد decoder فعلي لقراءة الباركود.
- `AppointmentsPage.tsx` يستخدم `window.location.reload()` بعد إنشاء الموعد، وهذا سلوك غير مثالي.
- `LandingPage.tsx` موجودة لكن التطبيق الأساسي `src/app/App.tsx` لا يربطها ضمن routes الحالية.
- توجد نسخة بديلة `src/app/App_simple.tsx` تبدو غير مستخدمة.
- `ImageUpload.tsx` قد يفقد بعض الصور عند رفع عدة ملفات معاً بسبب تحديث الحالة بنمط stale state.

### 3) مشاكل أمن/اعتمادات
نتيجة `npm audit` أظهرت 13 vulnerability:
- High: `react-router-dom`, `vite`, `ws`
- Moderate: `esbuild`, `js-yaml`, `postcss`
- Low: `eslint`, `@babel/core`, `@supabase/supabase-js` وغيرها

## خطة التنفيذ الفورية
1. إصلاح lint بالكامل.
2. تشغيل lint مرة ثانية للتأكد.
3. إعداد تقرير تدقيق تفصيلي بالمشاكل المكتشفة.
4. بدء إصلاحات low-risk/high-value التالية:
   - إزالة `window.location.reload()` من المواعيد.
   - تحسين `ImageUpload` لمعالجة رفع عدة صور بشكل صحيح.
   - توضيح/توثيق أن barcode scanner الحالي ليس scanner فعلياً أو ترقيته لاحقاً.
5. بعد ذلك مراجعة routing وربط Landing Page حسب هدف المنتج.

## ناتج متوقع من المرحلة الحالية
- تقرير تدقيق منظم.
- إصلاحات سريعة بدون كسر البناء.
- قائمة واضحة لما هو جاهز للبيع وما يحتاج مرحلة ثانية.
