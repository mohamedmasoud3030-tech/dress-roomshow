## المشاكل المكتشفة التي ما زالت قائمة

## High Priority

### 1) Barcode Scanner غير مكتمل وظيفياً
**الملف:** `src/features/dresses/BarcodeScanner.tsx`
- الكود يفتح الكاميرا فقط
- لا توجد مكتبة decoding فعلية لقراءة الباركود
- لا يوجد loop للتعرف على frames أو parsing للرمز

**الأثر:**
- الميزة المعلنة كـ "barcode scanner" ليست scanner حقيقي حتى الآن

**التوصية:**
- دمج مكتبة مثل `@zxing/browser` أو `html5-qrcode`
- أو إعادة تسمية الميزة مؤقتاً لتكون "كاميرا + إدخال يدوي" حتى لا يحدث overclaim

### 2) Landing Page غير مدمجة في التطبيق الرئيسي
**الملف:** `src/pages/landing/LandingPage.tsx`
- الصفحة موجودة
- لكن `src/app/App.tsx` لا يربطها حالياً
- توجد نسخة `App_simple.tsx` تبدو تجريبية/بديلة وغير مستخدمة في bootstrap الحالي

**الأثر:**
- الميزة موجودة ككود لكن ليست ضمن تجربة التطبيق الرئيسية الحالية

**التوصية:**
- تحديد هل المطلوب:
  1. جعل Landing Page هي الصفحة الرئيسية العامة
  2. أو جعلها route مستقل مثل `/landing`
  3. أو فصلها كتجربة عميل مستقلة

### 3) رفع الصور باستخدام Base64 داخل localStorage
**الأثر:**
- سريع للتجربة
- لكنه سيكبر بسرعة مع 5 صور لكل فستان
- قد يسبب:
  - استهلاك quota
  - بطء في القراءة/الحفظ
  - تضخم snapshot/storage

**التوصية:**
- لو التطبيق local-first فقط: التفكير في IndexedDB للصور
- أو في نسخة Tauri: حفظ الملفات محلياً والاحتفاظ بالمسارات فقط

### تم تنفيذ دفعة ثانية

2. تم عرض الصور الفعلية في `DressesPage`
3. تم ربط Landing Page كمسار فعلي داخل التطبيق على `/landing`
4. تم تحسين Landing Page بإضافة فلترة حسب الفئة وربط واضح إلى صفحة المواعيد

### تم تنفيذ دفعة ثالثة
1. تم تحويل `BarcodeScanner.tsx` من واجهة كاميرا فقط إلى ماسح فعلي باستخدام `@zxing/browser`
2. تم دعم القراءة التلقائية للباركود من الكاميرا مع fallback للإدخال اليدوي
3. تم ضبط إيقاف الكاميرا والتنظيف عند الإغلاق أو بعد نجاح المسح
4. تم الإبقاء على التحقق الكامل: `typecheck`, `lint`, `build` كلها ناجحة بعد الدمج

### ملاحظات على التنفيذ الحالي للباركود
- تم تقييد الصيغ المستهدفة إلى `CODE_128`, `EAN_13`, `EAN_8`
- ما زال يلزم اختبار runtime فعلي على جهاز بكاميرا للتأكد من جودة التجربة وسرعة الالتقاط
- مكتبة `@zxing/library` أظهرت تحذير engine أثناء التثبيت في بيئة Node الحالية، لكن البناء والأنواع نجحوا محلياً

### تم تنفيذ دفعة رابعة
1. تم ربط `BarcodeScanner` داخل `DressesPage`
2. تمت إضافة زر `مسح باركود` داخل واجهة المخزون
3. عند نجاح المسح يتم البحث عن الفستان عبر `barcode` أو `code`
4. يتم تطبيق فلتر بحث تلقائي على الفستان المكتشف مع تمييزه بصرياً داخل النتائج
5. تم التحقق مجدداً من `typecheck`, `lint`, `build` بنجاح

### ملاحظات أداء جديدة
- بعد دمج مكتبة المسح أصبح bundle الإنتاجي أكبر بشكل ملحوظ
- ظهر تحذير Vite بأن chunk رئيسي تجاوز 500 kB بعد minification
- هذا ليس build blocker حالياً، لكنه يستحق معالجة في خطوة لاحقة عبر lazy loading أو code splitting

### الدفعة القادمة المقترحة
1. تحسين `BarcodeGenerator` من ناحية طباعة الملصق وإدارة `Blob URL`
2. فصل `BarcodeScanner` lazy-loaded لتقليل حجم الـ bundle الرئيسي
3. وضع حدود/تحذيرات لحجم الصور المخزنة في localStorage
4. عمل dependency upgrade branch آمن
