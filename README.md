# LENA — Dress Roomshow

تطبيق عربي RTL لإدارة وتشغيل معرض واحد لفساتين ومستلزمات المناسبات، بنمط Local-first.

## هدف المنظومة

LENA يربط الحقيقة المادية والتجارية والمالية والتشغيلية للمعرض في نظام واحد:

- حالة كل عنصر ومكانه ومدى جاهزيته.
- العميلة والحجز والمواعيد والتسليم والاسترجاع.
- التحصيل والعربون والغرامات والاسترداد والبيع والمصروفات.
- الفحص والغسيل والصيانة بعد الإرجاع.
- إقفال اليومية والتقارير وسجل التدقيق.
- النسخ الاحتياطي والاستعادة وتشغيل PWA وTauri Windows.

تفاصيل منطق الأعمال وحالات العناصر والحجوزات والحركات المالية موجودة في [`docs/BUSINESS_MODEL.md`](docs/BUSINESS_MODEL.md).

## نطاق المنتج

- المخزون بأنواعه: فساتين، إكسسوارات، حقائب، أحذية، طرح، وأخرى.
- العملاء والحجوزات والمواعيد والتسليم والاسترجاع.
- المدفوعات والعربون والغرامات والمصروفات والبيع المباشر.
- التقارير وإقفال اليومية وسجل التدقيق والنسخ الاحتياطي.
- منع تعارض الحجوزات وفترة تجهيز قابلة للضبط.
- صفحة عرض عامة للمخزون قابلة للتخصيص.

النسخة الحالية مخصصة لمعرض واحد فقط. لا تشمل SaaS أو تعدد الفروع/المستخدمين أو مزامنة الأجهزة أو الدفع الإلكتروني.

## التخزين والتشغيل

- الويب وPWA يستخدمان تخزينًا محليًا داخل المتصفح.
- نسخة Tauri تحفظ Snapshot محليًا داخل SQLite وتستعيده عند التشغيل.
- بيانات التشغيل يجب أن تمر عبر عقد التخزين المركزي حتى تدخل النسخ الاحتياطي والاستعادة وإعادة الضبط ونسخة سطح المكتب.

## التقنية

React، TypeScript، Vite، Tailwind CSS، React Router، PWA، Tauri 2، وSQLite.

## حالة الإطلاق

`main` يحتوي تطبيقًا تشغيليًا، لكنه ليس Release نهائيًا. بوابات التسليم الحالية تشمل خصوصًا:

- نقل المخزون والمواعيد إلى عقد التخزين الموحد بدون فقد البيانات القديمة.
- إدخال الصور والفواتير والمرتجعات وكل الكيانات في النسخ الاحتياطي وSnapshot سطح المكتب.
- جعل عمليات الدفع والتسليم والاسترجاع والبيع متعددة الملفات Atomic أو Compensation-safe.
- فصل التدفق النقدي عن الإيراد والعربون القابل للاسترداد في التقارير.
- منع الحذف وإعادة استخدام الأكواد بعد وجود تاريخ تشغيلي.
- اختبار PWA وTauri Windows وتدفقات الهاتف فعليًا.
