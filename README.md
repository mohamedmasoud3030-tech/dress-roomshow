# dress-roomshow

تطبيق إدارة محل فساتين (Dress Roomshow) بواجهة عربية RTL أولاً لإدارة المخزون والحجوزات والعملاء ضمن نطاق البدء المعتمد.

## هدف المشروع
- تنظيم تشغيل المحل اليومي عبر لوحات بسيطة وواضحة.
- إدارة الفساتين (الحالة، الفئة، الأسعار).
- متابعة العملاء وأرصدتهم.
- متابعة الحجوزات ومواعيد الاستلام/الإرجاع.

## نطاق البدء المعتمد
- صفحات أساسية: Dashboard, Dresses, Customers, Reservations.
- صفحات مستقبلية Placeholder فقط: التسليم والاسترجاع، المدفوعات، المصروفات، التقارير.
- الاعتماد الحالي على بيانات Mock محلية بدون ربط قاعدة بيانات إنتاجية.

## التقنية المستخدمة
- React + TypeScript
- Vite
- Tailwind CSS + PostCSS + Autoprefixer
- React Router
- (مهيأ) Tauri لتغليف سطح المكتب

## الأوامر المتاحة
- `npm install` تثبيت الاعتمادات
- `npm run dev` تشغيل بيئة التطوير
- `npm run lint` فحص الجودة عبر ESLint
- `npm run build` بناء نسخة الإنتاج
- `npm run preview` معاينة نسخة البناء

## وضع البيانات الحالي (Mock Data)
- البيانات الحالية تجريبية وموجودة داخل ملفات Features.
- هذا مفيد للتطوير السريع ولكنه ليس مصدر بيانات مشترك متعدد المستخدمين.
- لا يوجد حالياً مزامنة لحظية أو صلاحيات مستخدمين أو سجل تدقيق.

## الخطوة التالية
- دمج Supabase كمصدر بيانات فعلي (Auth + Database + Storage) مع الحفاظ على نفس نماذج الدومين قدر الإمكان.

## ملاحظات PWA و Tauri
- تم توفير أصول PWA الأساسية (`favicon` و`pwa icons`) بشكل Placeholder.
- إعدادات PWA الحالية مناسبة للتطوير الأولي وتحتاج تحسين بصري قبل الإطلاق.
- مشروع Tauri موجود للتغليف المكتبي، ويمكن تفعيله لاحقاً بعد تثبيت تدفق الويب والبيانات.
